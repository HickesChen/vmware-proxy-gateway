# 变更日志

[English](CHANGELOG.md)

## 0.1.10

- 增加 Shell 代理生命周期管理、旧 VS Code Server 环境检测、诊断自动修复和面向用户的故障原因与解决方案。
- 强化关闭代理、安全退出和卸载清理，覆盖 Shell、TUN/DNS、nftables、自动启动、用户配置以及本软件安装的 sing-box，并在卸载后复验残留。
- 增加 Noble/amd64 离线运行资源、应用私有 Python 依赖、SHA-256 校验，以及平台不匹配时的最小下载回退；目标机不再使用 pip。
- 场景验证扩展到 23 项。

## 0.1.5

- 强化运行时配置处理：代理主机为域名、代理端口非法、自定义绕过域名规范化、服务运行中重启生效、残留 TUN/DNS 清理、UDP 阻止开关和默认 IPv4-only DNS 行为。
- 强化安装器：当系统已有 sing-box 但不在 `/usr/local/bin` 时会同步到实际服务路径，并增加 sing-box 校验。
- 强化发布自动化：支持带未提交变更的发版工作区、检测远程分支分叉、Release 附件覆盖上传，以及显式安装发布 workflow 依赖。
- 场景验证从 9 项扩展到 10 项。

## 0.1.4

- 添加橘黄色开启态托盘图标，用户可以直接从系统托盘区分代理开启/关闭状态。
- 添加开启态托盘图标染色逻辑的验证覆盖。

## 0.1.3

- 添加 English、简体中文、繁體中文 GUI 语言选择。
- 添加桌面图标和系统托盘图标。
- 添加托盘菜单操作：打开、应用、开启、关闭、测试、诊断、刷新和安全退出。
- 添加安全退出：GUI 退出前先停止代理服务。
- 添加 GUI 单实例锁。
- 添加系统包下载和容器镜像仓库预设绕过开关。
- 添加动态 APT 源域名检测，支持官方源、镜像源和私有源。
- 添加常见包管理器的进程级直连规则，让 APT/Snap/Flatpak 下载不再只依赖域名识别。
- 在启动/重启服务前，如果能找到用户配置，会先刷新生成的服务配置。
- 添加更稳健的本地 DNS 检测，不再假设 `127.0.0.1:53` 可用。
- 添加场景验证脚本。
- 添加适合受限或离线环境的安装器选项。
